一条命令，五件事自动完成

没有参数，没有配置文件。CloneFaster 从零走到全部仓库就绪，中间零交互。

🔍

检测 git

自动检查 git 是否已安装。缺失则按操作系统输出精确安装命令。

🔑

检测 gh CLI

自动检查 GitHub CLI 是否存在。macOS / Debian / Windows 各有对应的引导安装命令。

🌐

自动登录

gh CLI 未登录时，自动唤起浏览器执行 gh auth login --web。无需记忆任何命令。

📋

拉取仓库列表

从 GitHub API 分页拉取当前用户/组织的全部仓库——公开和私有，全部自动合并去重。

⚡

并行克隆

ThreadPoolExecutor 并行调度（默认 10 仓库），每仓库 --depth 1 --single-branch 浅克隆。已存在且完整的仓库自动跳过。

极简到只剩下一个文件

整个 CloneFaster 就是一个独立的 Python 脚本。零外部包依赖，纯标准库。

# CloneFaster 依赖清单
import argparse, base64, json, os, platform, re, shutil
import signal, subprocess, sys, threading
import urllib.error, urllib.request
from concurrent.futures import ThreadPoolExecutor
from pathlib import Path

# 就是这么多了。没有 requests，没有 pygithub，没有 keyring。
# pip install 不需要，因为根本没依赖。
设计决策做法原因
认证gh auth tokengh CLI 是 GitHub 用户的标配工具，CloneFaster 不自己实现 OAuth
网络请求urllib.requestPython 3 自带，零外部 Python 依赖
克隆模式--depth 1 --single-branch批量同步只需要最新快照
去重git fsck --strict已存在且完整的仓库直接跳过
协议选择SSH → HTTPS 自动回退有 SSH 用免密 SSH，没有则走 token 认证的 HTTPS
并行模型ThreadPoolExecutorgit clone 是子进程阻塞调用，线程池恰好匹配 I/O 密集型任务
分发方式curl | python3 -零摩擦——不需要 pip / pipx / uv / 任何包管理器
进度反馈单行原地刷新进度条自适应终端宽度，非 TTY 自动关闭——和 apt / npm 一致
